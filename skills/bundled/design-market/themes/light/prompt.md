# Brief Handoff / Light Theme

Brief Handoff 是一个明亮办公协作主题，用于把团队 brief、审批状态、会议节奏、交付风险和交接清单组织成可直接落地的业务网站。

## 审美 DNA

- Pearl Workspace / quiet daylight productivity：白色主表面、浅米绿色画布、青灰主强调 `#526f7b`、温金辅助强调 `#a77720`。
- 低噪声边界、细腻阴影、清晰层级、克制圆角；像真实办公软件，不像营销空壳。
- 信息密度要高但不压迫：状态、负责人、截止时间、风险提示必须可读。
- 动效只用于反馈 hover / focus / active，幅度低、时间短，并支持 `prefers-reduced-motion`。

## 使用规则

1. 这不是组件库。不要把按钮、卡片、表格孤立复制后堆在页面上。
2. 必须结合业务。每个区块都要解释真实流程：谁在交接、交接什么、什么状态、下一步动作是什么。
3. 禁止直接复制。可以借鉴布局节奏、色彩和状态表达，但必须按目标业务重写文案、指标和信息结构。
4. 禁止堆叠组件。页面应该像完整产品体验，有导航、主张、工作流、风险提示、清单和行动闭环。
5. 禁止 Lorem ipsum、空洞口号、随机渐变、厚重发光、低对比文字、无语义状态色。

## 推荐场景

- 项目启动 brief 中心
- 跨团队交付 handoff 页面
- 办公 SaaS dashboard / workspace
- 设计、运营、产品、销售之间的协作状态页

## 生成要求

- 纯前端 HTML + 内联 CSS，不依赖远程资源。
- 响应式布局，移动端不横向溢出。
- 表单控件必须有 label 或可访问名称。
- 状态色必须服务真实语义，例如 ready、blocked、needs review、due today。
- 按钮与链接必须有 hover、focus-visible、active 状态。
