# Soft Instrument Control Room

为真实业务设计一个“柔和仪器面板 / 参数控制”网站，而不是展示一组孤立控件。视觉 DNA 来自新拟物 Soft Instrument：浅灰蓝基底、右下暗影与左上亮影、凹陷轨道、凸起按钮、低对比但清晰的信息层级。

## 必须呈现的业务语境

- 选择一个明确业务：设备校准、音频母带调校、实验室阈值控制、能耗策略控制或机器人参数巡检。
- 页面必须像完整产品网站：导航、首屏主张、实时状态摘要、参数调节区、任务队列 / 数据表、说明或转化区。
- 文案、指标、状态、表格行、输入值必须是业务相关内容，禁止 Lorem ipsum、空洞口号和纯装饰数字。

## 视觉与交互要求

- 基底使用 #e8ecf0 / #e6ece8 一类柔和浅灰蓝，不要纯白背景。
- 组件边界主要依靠双向阴影：右下柔和暗影、左上柔和亮影；输入、滑轨和局部面板可使用 inset 阴影表现凹陷。
- 主强调色使用 #526f7b，辅助强调可使用 #9b7c33；状态色必须有真实语义，不要随机上色。
- 保持低噪声、低幅度、低频率动效；hover、focus、active 状态要完整，且支持 prefers-reduced-motion 降级。
- 移动端必须响应式，不能横向溢出；密集数据在小屏可转为卡片式行。

## 禁止事项

- 不是组件库：不要只输出按钮、卡片、toggle、slider 的陈列页。
- 必须结合业务：每个模块都要说明或推动具体业务流程，不能做无上下文 UI 展示。
- 禁止直接复制：不要照搬旧 theme-neumorphism.json 中的结构、文案或代码，只保留审美 DNA 后重新组织完整网站。
- 禁止堆叠组件：不要把多个新拟物控件机械堆在一起；布局必须有主次、流程和叙事。
- 禁止厚重发光、廉价大渐变、过度圆角、低对比正文、无意义图标和只有“高级感”的视觉噱头。

## 推荐输出

生成完整纯前端页面，内联 CSS，无远程资源。优先使用语义 HTML、系统字体、可访问标签、真实按钮和表格结构。整体应该像一个可上线的参数控制产品首页，而不是素材拼贴。
