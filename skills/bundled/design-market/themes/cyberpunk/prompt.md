# Signal Router Telemetry

## 主题定位

这是一个面向网络遥测 / 路由基础设施业务的完整网站主题，不是组件库。它应该像一个真实的骨干网运维控制台官网：展示路由健康、链路延迟、丢包、边缘节点、故障域和处置动作，而不是把按钮、卡片、表格堆在一起。

## 审美 DNA

- 深色工程表面：以 `#060817`、`#090b1a` 为主，避免廉价纯黑大面积压暗。
- 精密网格：使用低透明度冷蓝细线、透视网格、坐标刻度和扫描线表达基础设施感。
- 克制霓虹：主强调 `#22d3ee`，辅助 `#ff2bd6` 只用于告警、路径高亮或关键动作，禁止满屏厚重发光。
- 数据优先：所有装饰都必须服务路由、节点、状态和判断，不做随机赛博符号。
- 终端质感：可局部使用等宽字体、短标签、时间戳、ASN、POP、BGP、p95、packet loss 等真实网络语义。

## 业务场景

适合生成以下类型的完整页面：

- 网络可观测平台官网
- 路由调度 / 流量工程控制台介绍页
- 边缘节点健康监控页面
- 骨干网 SLA 与实时遥测展示页
- NOC 团队事故响应工作台

## 必须包含

- 一个明确的业务主张，说明该产品如何提升路由可靠性或故障定位效率。
- 真实网络遥测指标，例如 p95 latency、packet loss、route convergence、active POPs、BGP sessions、incident MTTR。
- 至少一个可读的路由或节点状态区域，状态色必须有语义。
- 明确主动作和次动作，例如 “Trace route”、“Inspect incidents”、“Open NOC view”。
- 响应式布局：桌面可多列，移动端必须单列且无横向溢出。
- 完整纯前端实现：单个 HTML 文件、内联 CSS、无远程资源、无外部字体、无图片依赖。

## 禁止事项

- 不是组件库：不要输出孤立 button/card/table/nav 的集合。
- 必须结合业务：不要只写“高级感”“未来感”“科技感”，必须落到网络遥测与路由基础设施。
- 禁止直接复制本参考站或旧素材代码；只能继承审美 DNA、信息层级和业务方向。
- 禁止堆叠组件：页面必须有完整叙事、连续视觉节奏和清晰信息架构。
- 禁止 Lorem ipsum、空洞口号、随机装饰字符、无意义霓虹、过度毛刺、低对比文字。
- 禁止远程 CSS、远程 JS、远程图片、iframe 或第三方资源。

## 生成建议

先定义业务对象：谁在用、监控哪些链路、最危险的异常是什么、用户下一步要做什么。再组织页面：首屏主张、实时拓扑/遥测、节点健康、告警队列、处置流程、可信指标。视觉层面保持冷峻、精密、低噪声，让霓虹像仪表信号而不是装饰灯牌。
