# Index Research Journal

为数字杂志、研究报告、年度洞察、行业白皮书发布页生成完整网站。视觉 DNA 来自出版级 Index Magazine：暖白纸面、深墨文字、砖红强调、极细分隔线、非对称网格、衬线标题与无衬线正文混排。页面应像一份可正式发布的研究刊物，而不是演示稿或素材拼贴。

## 必须遵守

- 这不是组件库：不要输出孤立 card、button、nav 或 section 的堆叠清单，必须生成一个从首屏到页脚完整连贯的网站。
- 必须结合业务：围绕具体报告主题、真实目标读者、研究方法、关键结论、下载/订阅/预约等业务动作组织内容。
- 禁止直接复制：不得直接复制参考素材、旧 JSON 片段、现有 HTML/CSS 或示例文案；只能继承审美 DNA 与信息组织原则。
- 禁止堆叠组件：每个区块必须承担明确叙事功能，区块之间要有阅读节奏、证据递进和转化路径。

## 网站结构建议

1. 顶部出版导航：刊名、期号、章节入口、主动作。
2. 杂志式首屏：报告标题、发布日期、研究范围、编辑摘要、关键动作。
3. Executive brief：用短段落和少量数据说明核心判断。
4. Evidence board：展示样本量、增长率、风险信号、置信等级等真实指标。
5. Chapter index：像目录一样呈现章节、页码、阅读时长和状态。
6. Featured insight：用大标题、引文、图表占位和说明文字突出一个关键洞察。
7. Methodology：说明数据来源、研究周期、访谈对象和限制条件。
8. Conversion footer：提供下载报告、订阅更新、预约解读或联系研究团队。

## 设计原则

- 使用暖白纸面 `#fbfaf7`、纸张浅灰 `#f1ede4`、深墨 `#151515`、砖红 `#b03d2d`。
- 以细线、留白、字号对比、列宽和编号建立层级，不依赖厚重阴影、廉价渐变或发光。
- 标题可以使用 Georgia / Times New Roman 等系统衬线字体；正文使用 Inter / system-ui。
- 布局应偏出版物：非对称分栏、章节编号、边注、目录、图注、pull quote、数据表。
- 所有内容必须像真实业务：不要 Lorem ipsum，不要空洞口号，不要没有来源的夸张承诺。
- 交互状态要完整：链接、按钮、表单应有 hover/focus/active；支持 `prefers-reduced-motion`。
- 必须响应式：窄屏改为单列，表格或目录不能横向溢出。
- 完整纯前端：内联 CSS，无远程字体、图片、脚本或第三方资源。

## 禁忌

- 不要把页面做成通用 SaaS dashboard。
- 不要用随机渐变、玻璃拟态、厚圆角、强发光或装饰性噪点掩盖信息层级。
- 不要只复刻 Monocle/Kinfolk 风格的表面排版；要服务研究发布、阅读、引用和转化。
- 不要输出无法落地的概念稿；HTML 结构必须语义清晰，可直接预览。
