# Living Habit Field / Organic Theme

这是一个面向健康生活方式与习惯追踪业务的完整网站主题，用于生成可直接落地的产品参考站，而不是生成孤立 UI 片段。

## 业务定位

- 场景：个人健康管理、生活方式教练、习惯追踪、晨间/睡前节律、补水、睡眠、步行、情绪日志。
- 核心目标：帮助用户理解“今天下一步该做什么”，而不是制造装饰性的自然氛围。
- 信息结构：首屏主张、今日完成度、身体信号指标、一周习惯日志、教练建议、月度连续记录。

## 审美 DNA

- 参考旧 `theme-organic.json` 的 Living Field 气质：暖纸表面、自然绿主强调、陶土棕辅助强调、低对比纹理、轻阴影、柔和不规则圆角、低幅度呼吸动效。
- 主色建议：`#f5f3e7`、`#fdfbf7`、`#dfe7d3`、`#233128`、`#4f7b58`、`#b08345`。
- 字体节奏：标题可用 Georgia / serif 营造自然编辑感；正文使用 system sans-serif 保持产品可读性。
- 动效只用于节律表达：呼吸环、轻微 hover、缓慢 blob 变化；必须支持 `prefers-reduced-motion` 降级。

## 必须做到

1. 必须结合具体健康生活方式 / 习惯追踪业务生成页面，包含真实文案、真实指标、真实状态和真实任务流。
2. 必须输出完整纯前端页面：HTML + 内联 CSS；不依赖远程图片、远程字体、外部 JS 或第三方资源。
3. 必须响应式适配桌面、平板、手机，移动端不能横向溢出。
4. 必须有清晰 hover / focus / active 状态，按钮和输入控件具备可访问焦点。
5. 必须保证状态色服务语义，例如完成、待记录、需观察、稳定趋势。
6. 必须用布局讲清楚产品，而不是把卡片随意摆满页面。

## 严格禁止

- 不是组件库：不要只输出按钮、卡片、表格等孤立组件清单。
- 必须结合业务：不要生成抽象的“自然风”“健康感”页面，所有区块都要服务习惯追踪业务。
- 禁止直接复制：不要直接复制旧 `theme-organic.json` 的 HTML/CSS 或文案；只能继承审美 DNA 并重新设计完整网站。
- 禁止堆叠组件：不要把导航、hero、卡片、表格机械堆叠；每个区块必须形成一个连续的用户任务流。
- 禁止 Lorem ipsum、空洞口号、随机叶子装饰、廉价大渐变、厚重发光、低对比正文、过度圆角和无语义状态色。
- 禁止远程资源、外链图片、CDN 字体、依赖框架脚本。

## 生成检查

生成结果应像一个真实健康习惯产品的营销 + 工作台首页。用户看完应能判断：今天完成了什么、还差什么、为什么建议这样做、下一步点击哪里。
