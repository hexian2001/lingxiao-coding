# 极简 SaaS 产品决策台

你要生成一个完整的网站页面，而不是孤立的 UI 片段。主题是「极简 SaaS 产品决策台」：给产品负责人、增长团队或运营团队查看关键风险、发布状态、实验优先级和下一步动作。

## 审美 DNA

- 使用克制、近纸感的浅色表面：`#f7f8f5`、`#ecefeb`。
- 主文字与关键动作使用接近黑色的 `#111719` / `#151b1d`。
- 只使用一个冷静的辅助强调色 `#526f7b`，用于状态、趋势或选中项。
- 信息层级优先于装饰：细边框、低阴影、小圆角、留白、对齐、表格化数字。
- 动效必须低幅度、低频率，并支持 `prefers-reduced-motion` 降级。

## 内容要求

- 必须结合具体业务场景，例如产品发布决策、客户流失风险、功能实验排期、收入质量监控。
- 页面需要包含真实语义：指标、状态、表格行、负责人、日期、判断理由、主动作。
- 文案必须像真实 SaaS 工作台，不使用 Lorem ipsum，不写空洞口号。
- 首屏需要让用户立刻知道：当前要判断什么、风险在哪里、下一步该做什么。

## 强制约束

- 这不是组件库：不要输出一堆可复用组件说明或组件拼盘，要输出一个完整、连贯、可访问的网站页面。
- 必须结合业务：每个区域都要服务一个明确业务判断，不能只展示视觉效果。
- 禁止直接复制：不要照搬参考素材中的 HTML、CSS、文案或布局，只继承其克制、清晰、低噪声的审美 DNA。
- 禁止堆叠组件：不要把导航、卡片、表格、表单机械堆叠；要让页面结构围绕一个产品决策流程展开。
- 禁止廉价大渐变、厚重发光、随机装饰、过度圆角、低对比文字和无语义状态色。

## 工程要求

- 纯前端实现，可以直接保存为单个 HTML 文件运行。
- CSS 必须内联在页面中，不依赖外部样式表。
- 不使用远程字体、图片、图标库、脚本或 CDN。
- 必须响应式：桌面端是清晰工作台，移动端单列阅读且不横向溢出。
- 交互元素需要 hover/focus/active 状态，键盘焦点可见。
