# Gradient Theme — Chromatic AI Engine

## 主题定位

这是一个完整的 AI 创意工具 / 色彩生成引擎网站主题，用于呈现“从业务 brief 到可解释色彩系统”的真实产品体验。它适合品牌设计平台、AI 视觉生成工具、创意资产管理台、营销物料生成器和色彩策略 SaaS。

## 审美 DNA

- 继承旧 `theme-gradient.json` 的 Chromatic Engine 方向：精密色彩计算、深色底、半透明玻璃表面、克制渐变边界、低频流动感。
- 主强调色使用暖金 `#f2c673`，辅助使用冷灰蓝 `#aebdc5`，并以蓝 / 紫 / 青 / 品红作为 AI 生成和色谱状态的次级信号。
- 渐变不能只是背景装饰，必须表达业务含义：生成中、匹配度、色彩和谐度、品牌偏移、可访问性风险或导出状态。
- 视觉质感应像专业创意工具，不像活动页换皮：信息密度高但有秩序，发光克制，圆角适中，动效缓慢且可降级。

## 必须包含的业务结构

生成页面时要把主题落成完整网站，而不是孤立片段。至少包含：

1. 顶部导航：品牌、核心入口、当前生成状态和主操作。
2. Hero：一句清晰产品主张、真实业务说明、两个明确 CTA。
3. AI Brief 输入区：包含真实输入内容、约束标签、生成按钮和模型状态。
4. Palette Engine 工作区：展示多组色彩结果、和谐度评分、对比度检查、品牌偏移说明。
5. 生成队列 / 任务表：包含项目名、状态、评分、更新时间或导出动作。
6. 应用场景区：展示生成色彩如何落到海报、界面、品牌资产或营销视觉。
7. 底部收束 CTA 或导出面板。

## 内容要求

- 文案必须结合具体业务场景，例如“为春季新品 Campaign 生成高对比色板”“把品牌主色扩展为可访问 UI token”“为 12 张社媒视觉统一色彩情绪”。
- 指标必须真实可读，例如 harmony 96、contrast AA、brand drift 4.8%、queue 12 jobs、export tokens 28。
- 状态必须有语义：Ready、Rendering、Needs review、AA Passed、Drift warning 等。
- 禁止 Lorem ipsum、空泛口号、无业务含义的数字和只有装饰作用的标签。

## 工程约束

- 必须是完整纯前端页面，可直接作为 `preview.html` 打开。
- CSS 必须内联在 HTML 中，不引用远程字体、图片、脚本、CDN 或外部资源。
- 必须响应式：桌面可展示双栏 / 多栏工作台，移动端单列且不横向溢出。
- 必须补齐 hover、focus-visible、active 等交互状态。
- 动效要低幅度、低频率，并通过 `prefers-reduced-motion` 降级。
- 保持语义 HTML：nav、main、section、article、table、form、button、footer 等按职责使用。

## 严禁事项

- 不是组件库：不要输出一堆按钮、卡片、表格的孤立组件集合。
- 必须结合业务：所有模块都要服务“AI 创意工具 / 色彩生成引擎”的完整产品叙事。
- 禁止直接复制：不得直接复制旧素材的 HTML/CSS/文案，只能吸收其审美 DNA 后重新设计完整网站。
- 禁止堆叠组件：不要把多个无关组件上下堆放；页面需要有清晰的信息架构、任务流和业务转化路径。
- 禁止廉价大渐变、厚重发光、随机装饰、低对比文字、过度圆角和无意义装饰符号。
