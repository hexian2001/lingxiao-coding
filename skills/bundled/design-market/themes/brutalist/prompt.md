# Museum Archive Catalogue / Brutalist 主题提示词

你要生成一个完整的粗野主义美术馆档案 / 展览目录网站，而不是组件库、不是零散 UI 素材拼贴、不是通用落地页换皮。

## 业务场景

适用于美术馆、公共档案馆、艺术节、文化研究机构的真实信息站：

- 当前展览与展厅开放状态
- 馆藏登记号、藏品摘要、保存限制、研究备注
- 研究访问预约、阅览席位、复制/出版申请
- 展览目录、墙面文字、策展说明和公共开放信息

所有模块都必须结合业务信息，不允许只写抽象口号；必须结合业务路径、访问目标和真实馆藏/展览数据组织页面。

必须结合业务：围绕展览目录、馆藏登记、研究访问、开放状态和申请动作组织完整页面。

## 审美 DNA

参考旧 `theme-brutalist.json` 的 Museum Brutalist 气质，但必须重新设计成完整网站：

- 硬网格、粗黑边框、零圆角、偏移硬阴影
- 档案纸色 `#f8f1cf` / 陈旧纸色 `#ece3bd`
- 墨黑 `#111111` 作为结构线、背景块和主文字
- 印章红 `#e03b2f` 作为状态、批注、主动作和编号强调
- 超大压缩标题、等宽编号索引、登记表格、图章式状态
- 视觉强烈但服务信息层级，不做随机反设计噪声

## 必须满足

- 必须输出完整纯前端页面，可直接保存为 HTML 运行。
- CSS 必须内联在页面中。
- 禁止使用远程资源：不引入外链字体、图片、CDN、脚本或第三方样式。
- 必须响应式：桌面为硬网格，多列信息；移动端收敛为单列，不能横向溢出。
- 必须有真实业务内容：展览名称、藏品号、状态、表格行、研究说明、访问动作。
- 必须有清楚交互状态：hover、focus-visible、表单 focus、按钮 active/hover。
- 必须保证可读性：高对比、语义标题、表格结构、可键盘访问。

## 禁止事项

- 不是组件库：不要输出一堆孤立按钮、卡片、表单示例。
- 禁止直接复制旧素材代码或旧 `theme-brutalist.json` 中的片段；只能继承审美 DNA。
- 禁止堆叠组件：页面必须围绕一个业务目标组织，而不是把 hero/card/table/nav/form 简单堆起来。
- 禁止 Lorem ipsum、空洞营销语、无业务含义的数字。
- 禁止廉价大渐变、厚重发光、玻璃拟态、过度圆角、随机装饰符号。
- 禁止用 placeholder 冒充内容；输入框应给真实示例值或明确标签。

## 推荐页面结构

1. 顶部馆方导航：品牌、编号入口、主动作。
2. 首屏展览档案：超大标题、公开档案章、真实说明、行动按钮、实时摘要面板。
3. 档案统计：展厅、数字化比例、研究席位等指标。
4. 目录索引：固定/侧边编号导航，指向页面真实段落。
5. 当前展览：三组有展厅、日期、状态和说明的展览记录。
6. 馆藏登记表：至少三行真实藏品数据，包含 Accession、Object、Room、Status、Research Note。
7. 研究访问筛选：关键词、访问目的、提交动作。
8. 页脚：开放时间、联系入口、申请说明。

## 输出质量标准

最终结果应该像一个可上线的文化机构参考站：粗粝、克制、档案感强；每条边框和每个红色强调都对应真实信息或动作，而不是为了装饰存在。
