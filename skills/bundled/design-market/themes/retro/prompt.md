# Retro Memory Terminal

## 主题定位

这是一个完整网站级视觉参考，方向是“复古终端记忆库 / 日志系统”。它适合用于 Agent 长跑控制台、审计日志平台、知识记忆库、发布记录室、事件追踪中心等真实业务页面。

## 审美 DNA

- 使用温暖复古软件气质：深紫黑终端表面、琥珀主强调、暖橙辅助强调、米色正文。
- 允许轻微 CRT 扫描线、纸质档案标签、磁带编号、等宽字体节奏，但必须克制，不做廉价怀旧贴纸。
- 视觉重点是“可读的日志、可信的归档、稳定的状态判断”，不是装饰效果本身。
- 圆角、阴影和发光都要轻；优先边框、留白、表格密度、状态色和层级。

## 必须遵守

1. 不是组件库：不要把按钮、卡片、表格、导航孤立罗列；必须组织成一个有业务叙事的网站页面。
2. 必须结合业务：文案、指标、日志行、状态、筛选项和 CTA 都要服务具体产品场景，不能使用 Lorem ipsum 或空洞口号。
3. 禁止直接复制：可以学习色彩、材质和信息节奏，但不能照搬本参考站的布局、文案、数值或 DOM 结构。
4. 禁止堆叠组件：不要用“导航 + hero + 卡片墙 + 表格 + footer”的机械堆叠冒充设计；每个区域都要解释用户要完成的任务。

## 生成要求

- 产物必须是完整纯前端页面，可直接保存为单个 HTML 文件运行。
- CSS 必须内联，不依赖远程字体、图片、图标库或脚本资源。
- 必须响应式：窄屏下不可横向溢出，表格/日志可以转为卡片式或可读列表。
- hover、focus、active 状态要清晰；动效低幅度、低频率，并支持 prefers-reduced-motion 降级。
- 状态颜色必须有语义：正常、警告、等待、失败不能只靠颜色区分，要配合文字标签。
- 内容应像真实业务：包含归档批次、日志来源、保留策略、写入状态、回放入口、审计摘要等。

## 适用场景

- Agent memory / worknote / session archive 管理台
- DevOps 发布日志、事件回放和审计记录
- 数据同步流水线的长跑监控页面
- 企业内部知识归档或检索入口
